## Original Project (Module 1)
This project began as a simple Number Guessing Game for Module 1. The goal was to let users guess a secret number, with hints after each guess. The initial version, generated by AI, had bugs: hints were misleading, the secret number changed unexpectedly, and winning was impossible.

## Title and Summary
**Game Glitch Investigator** is an interactive web app where users try to guess a secret number. The project demonstrates how AI can be used to build, debug, and enhance a simple game. It now includes a Retrieval-Augmented Generation (RAG) feature: users can ask questions about the game, and the AI retrieves relevant info from the documentation to answer.

## Architecture Overview
The system consists of:
- A Streamlit UI for user interaction.
- Game logic and scoring functions.
- A Retriever that searches the README for relevant info to answer user questions.
- An Evaluator/Tester for automated and manual testing.
- Human feedback for further improvement.

**Data Flow:**  
User input (guess or question) → UI → Game Logic/Retriever → Output (game result or AI answer) → User.  
Testing and human review ensure reliability.

## Setup Instructions

3. Run the app:  
   `python -m streamlit run app.py`
4. (Optional) Run tests:  
   `pytest`

## Scoring
- **Win**: Earn `100 - 10 × (attempt_number + 1)` points, minimum 10. Guess earlier to earn more (e.g., win on attempt 1 = 80 pts, attempt 2 = 70 pts).
- **Too Low**: Lose 5 points.
- **Too High**: Gain 5 points on even-numbered attempts; lose 5 points on odd-numbered attempts.
- Points accumulate across all guesses in the session.

## Design Decisions
- Used Streamlit for rapid UI development and easy state management.
- Added a simple RAG feature for explainability and user support.
- Kept logic modular for easier testing and debugging.
- Trade-off: The retriever uses keyword matching for simplicity, not a full language model.

## Testing Summary
- Used pytest for automated testing of game logic.
- Manual testing via the UI to check state and hint correctness.
- Learned that clear separation of logic and UI makes testing easier.

## Reliability & Testing Results

- Automated tests: 3/3 unit tests for game logic pass (see tests/test_game_logic.py).
- Logging and error handling: All retrieval and game logic errors are logged; no runtime errors observed during manual testing.
- Human evaluation: The AI’s answers to questions about the game were reviewed and found accurate when relevant info was present in README.md.

**Summary:**
All core logic tests pass. The RAG feature reliably retrieves relevant info when keywords are present, but may miss context if the question is too vague. Overall, the system is robust for its intended use case.

## Responsible AI Reflection

**Limitations & Biases:**
- The RAG feature relies on simple keyword matching, so it may miss context or nuance in user questions. It cannot reason beyond what is present in the README, and may provide incomplete or irrelevant answers if the documentation is lacking.
- The game logic is deterministic and fair, but the AI’s explanations are only as good as the source material.

**Potential Misuse & Prevention:**
- The AI could be misused if someone intentionally adds misleading or harmful content to the README, as the retriever will surface that information. To prevent this, documentation should be reviewed and version-controlled, and user questions should be filtered for inappropriate content.

**Surprises During Testing:**
- I was surprised by how often vague questions returned “No relevant information found.” This highlighted the importance of clear documentation and user queries. The logging helped quickly identify and fix issues with file access and error handling.

**Collaboration with AI:**
- Helpful: The AI suggested modularizing the game logic and adding a retrieval feature for explainability, which made the code easier to test and extend.
- Flawed: The AI initially proposed using advanced NLP for retrieval, which was overkill for this project’s scope; a simple keyword search was more reliable and transparent.

## Reflection
This project taught me the importance of robust state management in Streamlit and the value of automated testing. Integrating AI features like RAG can make even simple apps more interactive and user-friendly. I learned to balance simplicity and functionality, and to use AI as a tool for both building and debugging code.
